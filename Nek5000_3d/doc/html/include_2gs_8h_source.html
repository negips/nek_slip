9  Gather/Scatter Library
10 
11  The code
12 
13  struct comm c; // see "comm.h"
14  slong id[n]; // the slong type is defined in "types.h"
15  ...
16  struct gs_data *g = gs_setup(id,n, &c, 0,gs_auto,1);
17 
18  defines a partition of the set of (processor, local index) pairs,
19  (p,i) \in S_j iff abs(id[i]) == j on processor p
20  That is, all (p,i) pairs are grouped together (in group S_j) that have the
21  same id (=j).
22  S_0 is treated specially --- it is ignored completely
23  (i.e., when id[i] == 0, local index i does not participate in any
24  gather/scatter operation
25  If id[i] on proc p is negative then the pair (p,i) is "flagged". This
26  determines the non-symmetric behavior. For the simpler, symmetric case,
27  all id's should be positive.
28 
29  The second to last argument to gs_setup is the method to use, one of
30  gs_pairwise, gs_crystal_router, gs_all_reduce, gs_auto
31  The method "gs_auto" tries ~10 runs of each and chooses the fastest.
32  For a single-use handle, it makes more sense to use "gs_crystal_router".
33 
34  When "g" is no longer needed, free it with
35 
36  gs_free(g);
37 
38  A basic gather/scatter operation is, e.g.,
39 
40  double v[n]; buffer buf; // see "mem.h" for "buffer"
41  ...
42  gs(v, gs_double,gs_add, 0, g,&buf);
43 
44  The buffer pointer can be null, in which case, a static buffer is used,
45  shared across all gs handles.
46  This gs call has the effect, (in the simple, symmetric, unflagged case)
47 
48  v[i] <-- \sum_{ (p,j) \in S_{id[i]} } v_(p) [j]
49 
50  where v_(p) [j] means v[j] on proc p. In other words, every v[i] is replaced
51  by the sum of all v[j]'s with the same id, given by id[i]. This accomplishes
52  "direct stiffness summation" corresponding to the action of QQ^T, where
53  "Q" is a boolean matrix that copies from a global vector (indexed by id)
54  to the local vectors indexed by (p,i) pairs.
55 
56  Summation on doubles is not the only operation and datatype supported. The
57  full list is defined in "gs_defs.h", and includes the operations
58  gs_add, gs_mul, gs_max, gs_min
59  and datatypes
60  gs_double, gs_float, gs_int, gs_long, gs_sint, gs_slong.
61  (The int and long types are the plain C types, whereas sint and slong
62  are defined in "types.h").
63 
64  For the nonsymmetric behavior, the "transpose" parameter is important:
65 
66  gs(v, gs_double,gs_add, transpose, g,&buf);
67 
68  When transpose == 0, any "flagged" (p,i) pairs (id[i] negative on p)
69  do not participate in the sum, but *do* still receive the sum on output.
70  As a special case, when only one (p,i) pair is unflagged per group this
71  corresponds to the rectangular "Q" matrix referred to above.
72 
73  When transpose == 1, the "flagged" (p,i) pairs *do* participate in the sum,
74  but do *not* get set on output. In the special case of only one unflagged
75  (p,i) pair, this corresponds to the transpose of "Q" referred to above.
